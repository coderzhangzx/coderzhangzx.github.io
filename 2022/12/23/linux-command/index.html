linux常用命令

发表于 2022-12-23 更新于 2022-12-26 分类于 linux

ps （英文全拼：process status）命令用于显示当前进程的状态，类似于 windows 的任务管理器。

  • ps -ef | grep xxx：查看xxx进程的状态

netstat ：得知整个 Linux 系统的网络情况

netstat [-acCeFghilMnNoprstuvVwx][-A<网络类型>][–ip]

  • -n或–numeric：直接使用IP地址，而不通过域名服务器

  • -p或–programs：显示正在使用Socket的pid和name

  • -l或–listening：显示监控中的服务器的Socket

  • -t或–tcp：显示TCP传输协议的连线状况。

  • -u或–udp：显示UDP传输协议的连线状况。

nginx相关

  • yum安装的nginx在/etc/nginx/目录下       

  • nginx：因为已经配置环境变量，直接nignx即可开启服务

  • ps -ef | grep nginx：查看nigix进程的状态，第一个是主进程

  • kill -QUIT 主进程号：先杀子进程，最后安全杀死主进程（此时再次重复ps -ef 应该只剩一个color=auto的nginx，不用管）

  • nginx -s reload：更改nginx配置后，重新加载

docker相关

  • systemctl start/stop docker：开启/关闭docker

  • docker start/stop/restart [imageName]：开启/停止/重启镜像

  • docker images：查看本地镜像

  • docker rm [containerID]：删除容器

  • dockers rmi [imageID]：删除镜像（如果删除运行过的image，必须先删除承载这个image的container，因为container会保留image的引用）

  • docker ps：列出所有正在运行的容器

  • docker build -t [imageName] [projectDir]：根据当前路径下的Dockerfile，将projectDir路径下的所有文件打包成image，名字是imageName

  • docker run -p 80:8080 -v /data:/api -d [dockerName]：使用镜像 dockerName，以后台模式启动一个容器,将容器的 8080 端口映射到主机的 80 端口,主机的目录 /data 映射到容器的 /api。

  • docker logs -f [container-id]：动态实时查看某个容器的日志

文件/文件夹相关

  • mkdir/rmdir dir_name：新建/删除文件夹

  • cp

    • -i：复制文件前确认，如果已存在，是否覆盖

    • -r：递归复制

    • cp file1 file2：复制文件

    • cp -r dir1 dir2（dir2已存在）：将dir1及内部所有内容复制到dir2下

    • cp -r dir1 dir2（dir2不存在）：将dir1内部所有内容复制到dir2下

    • cp -r dir1/* dir2（dir2已存在）：将dir1内部所有内容复制到dir2下

  • rm

    • -r：递归删除

    • -f：强制删除

    • rm -rf filename/dir：强制删除文件/非空文件夹

  • touch filename：新建文件

  • cat filename：查看文件内容

  • cat filename | grep xxx：检查文件中是否存在内容xxx，如果存在会标红打印出来，否则不展示

sudo su：进入root权限

vim /etc/profile：linux中的环境变量文件，更改过后使用source /etc/profile刷新
